What do these numbers mean?

Based from the nature of the applications monitored by dGrav, high "read" applications, or applications that just retrieve data from the server tends to have lower data gravity proportion. This pattern can be explained by the concept "read" applications just displays the data it gets from the server. There are no modifications or other operations that may cause the nature of data to change. In short, such applications have low footprint or does not tend to affect other entities around it.

On the other hand, high "write" applications have higher data gravity because it changes the nature of data. It may add large amount of data in the data server at any times. When it add or manipulates data, the changes might also affect other entities in the network. If the application with high data gravity needs to be migrated to other data servers, it might be difficult to do so because there are so many dependencies that the application needs to satisfy first. Also, if an application has high data gravity, the services and other application dependent on it might have problem if that application would be changed.

Data Gravity Formula

Data Mass

Mass of the whole database

Application Mass

Mass of the current application

Latency

This is the average latency from the Application to the Data in seconds

Average Request

This is the average number of requests from the Application to the Data per second

Average Request Size

This is the average size of each request from the Application to the Data measured in Megabytes

Bandwidth

This is the average useable bandwidth from the Application to the Data in Megabytes per second
